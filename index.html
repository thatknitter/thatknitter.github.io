How I Work

A Project From Start to Finish

  • Gathering Information

    "Data data data! I cannot build bricks without clay!" -Sherlock Holmes

    You can't start a project without all of the information. What does the client want to do with this item? What do they want out of their site? Are there any unique challenges they face? How do they want this hosted? Who are the major competitors and what are they doing? There are some of the necessary questions a developer needs to ask. If you don't ask, you can't build the most effective product.

  • Make Decisions

    "Stay committed to your decisions, but stay flexible in your approach." -Tony Robbins

    Now I make the decisions. I decide what frameworks will be the most effective and choose the most appropriate programming language. The design is finalized, and pictures are chosen. Databases are designed based on information gathered previously. We also decide how and where the application will be deployed. Analytics and its implementation is decided upon as well.

  • Write Code

    "You might not think that programmers are artists, but programming is an extremely creative profession. It's logic-based creativity." -John Romero

    This is my favorite part of the process. This is where all of the code that makes up whatever we're building happens. We start by building the tests all of the programming will be based off of. Test Driven Development (TDD) is vital to a site that functions correctly once deployed. Without the tests, the only time we find out if something works is once the application is deployed, and users are having problems. Once the tests all pass, we go forward with the design and the rest of the program.

  • Deployment

    "Software is a great combination between artistry and engineering." -Bill Gates

    Deployment is not a "One and Done" operation. Once a program is deployed, it needs to be monitored. While the testing can eliminate many bugs, it's not a perfect system. Users will find bugs and design flaws that were not thought of during development. Updates and fixes need to be continually created and deployed. This is where we utilize the analytics system to look at reports and usage. From that information, we can improve the application over time.

Knit Designer

Software Program

Knit Designer and Knit Designer Pro are in-progress. This is a software program developed for knitting pattern designers so they can resize their patterns effectively, eliminating the need to make multiple mock-ups. It also addresses the issue of each publishing house having its own size requirements that do not match anyone else's sizes. So now, you can reliably resize a pattern with the correct stitch counts, and submit it to more than one publisher quickly and easily.

    Languages Used:
  • C#
  • Currently being redone in Java
  • Date: February 2015
  • Category: Software

BloxBot

Educational Program

BloxBot was created during Hack Tennessee 7 and was originally intended to be a software program. Instead, I managed to assemble a dream team of education professionals and other brilliant individuals who realized that most of what we needed was already developed and open source. BloxBot is a program designed to teach STEM education via Legos and Arduino. Kids start in middle school making Lego robots. By the time they're graduating high school, they're machining their own parts and automating everything. This gives them skills for both the job market, and higher education.
